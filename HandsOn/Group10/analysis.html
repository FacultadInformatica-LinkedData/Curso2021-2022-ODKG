Analysis of the different datasets

The datasets we are going to use have all the same format. They have 31 attributes and all together they have 2115 rows. The datasets are:

  • Actividades Culturales y de Ocio Municipal en los próximos 100 días
  • Actividades gratuitas en Bibliotecas Municipales en los próximos 60 días
  • Agenda de actividades y eventos
  • Actividades gratuitas en Bibliotecas Municipales en los próximos 60 días
In the following an analysis of each of the columns corresponding to the different attributes will be done:

ID-EVENTO

This attribute refers to the ID event. Clustering was not possible. There are 202 choices. An example would be: "10657294".

TITULO

This attribute refers to title. At the beginning there were 1022 choices, but after clustering we reduced it to 1010.It's set as a string/text type. An example would be: "¡Abracadabra! ¿De qué monstruo se habla?".

PRECIO

This attribute refers to the price of the events. At the beginning there were 70 choices, but after clustering we reduced it to 50. It's set as a string/text type because some prices are divided by age, early ticket, online purchase, etc. Down below we can see some examples of the choices left:
  • Acceso libre
  • 18 euros, 4 euros, 3,50 euros
  • Acceso libre hasta completar aforo
  • Próximamente a la venta
  • Consultar programación
  • Precio taquilla: 12 euros, Anticipada online : 10 euros

    • GRATUITO

    This attribute it's a boolean defining if the event is free or not. Clustering was not possible in this attribute since there are only 2 choices:
  • 1(TRUE) : 1527 rows
  • 0(FALSE) : 588

    • Two of the rows that correspond to category 1, which means that should be free in the attribute described before as price are defined as 'Más información en breve'.Consequently this discrepancy will have to be analyzed

    LARGA-DURACION

    This attribute is a boolean defining if the events duration is for more than a day or not. There are only two categories and clustering was not possible:
  • 1(TRUE) : 743 rows
  • 0(FALSE) : 1372

    • DIAS-SEMANA

    This attribute refers to the days of the week in which the events take place. There are 32 choices of different combinations of days of the week, for example :
  • L,M,X,J
  • V,S,D
  • M,X,J,V,S,D

    • Additionally there are 1372 blank rows of events that are not more than one day long.

    DIAS-EXCLUIDOS

    This attribute represents the days that are excluded from the calendar because of national and autonomical holidays. There are 98 choices and this attribute couldn't be clustered. The format is "YYYY-MM-DD".

    FECHA

    This attribute refers to the beginning date of the event. Clustering was not possible. There are 187 choices. An example of beginning date would be: 2021-10-01 00:00:00.0

    FECHA-FIN

    This attribute refers to the end date of the event. Clustering was not possible. There are 157 choices. An example of end date would be: 2021-10-06 23:59:00.0

    HORA

    This attribute refers to the Time of the event. Clustering was not possible. There are 34 choices. 09:45 (4), 10:31 (2), 10:45 (2), 11:00 (27), 11:30 (2), 12:00 (84), 12:30 (23), 13:00 (11), 15:30 (3), 17:00 (7), 17:30 (42), 18:00 (132), 18:30 (67), 19:00 (6), 20:00 (2), (blank) (34).
